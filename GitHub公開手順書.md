# 日本史クイズアプリ GitHub Pages 公開手順書

このアプリを GitHub Pages で公開し、お子さんにリンクを共有するための手順です。
**パソコンの操作（コマンド）は不要**で、すべてブラウザ上のクリックだけで完了します。

---

## ⚠️ 最初に大事なこと

このアプリのフォルダ（`勉強用アプリ作成`）は、模試などの教材が入った大きなフォルダの中にあります。
**模試フォルダごと公開しないでください。** 公開するのは、このアプリのファイルだけです。

公開するファイル（`勉強用アプリ作成` フォルダの中身）：

```
index.html          ← アプリ本体
css/style.css        ← デザイン
js/app.js            ← プログラム
data/                ← 問題データ（manifest.json と各時代の .json）
```

---

## ステップ1：新しいリポジトリを作る

1. パソコンのブラウザで [https://github.com](https://github.com) を開き、ログインします。
2. 画面右上の「**＋**」マークをクリック →「**New repository**」を選びます。
3. 次のように設定します。
   - **Repository name**（リポジトリ名）：`nihonshi-quiz` など、好きな半角英数字
   - **Public**（公開）を選択 ← Pages で公開するには Public が必要です
   - 「Add a README file」のチェックは**入れても入れなくてもOK**
4. 緑色の「**Create repository**」ボタンを押します。

---

## ステップ2：ファイルをアップロードする

1. 作成したリポジトリの画面で、「**Add file**」→「**Upload files**」をクリックします。
2. パソコンのエクスプローラーで `勉強用アプリ作成` フォルダを開きます。
3. **フォルダの中身**（`index.html` と `css` `js` `data` の各フォルダ）を、
   ブラウザのアップロード画面に**ドラッグ＆ドロップ**します。
   - 💡 `css` `js` `data` のフォルダごとドラッグすると、中のファイルも一緒に上がります。
   - 💡 `index.html` がリポジトリの**一番上（ルート）**に来るようにします。
     （`勉強用アプリ作成` というフォルダごと上げてしまうと、リンクが一段深くなります。中身だけ上げてください）
4. 画面下の「**Commit changes**」ボタンを押します。

アップロード後、リポジトリのトップに `index.html` `css` `js` `data` が並んでいればOKです。

---

## ステップ3：GitHub Pages を有効にする

1. リポジトリ上部のメニューから「**Settings**（設定）」をクリックします。
2. 左側のメニューから「**Pages**」をクリックします。
3. 「Build and deployment」の「**Source**」が「**Deploy from a branch**」になっていることを確認します。
4. 「Branch」のところで「**main**」（または `master`）を選び、フォルダは「**/ (root)**」のまま「**Save**」を押します。
5. 1〜2分待つと、ページ上部に公開URLが表示されます。

```
https://（あなたのユーザー名）.github.io/nihonshi-quiz/
```

---

## ステップ4：動作確認とリンク共有

1. 表示されたURLをクリックして、アプリが開くか確認します。
   - すぐに表示されない場合は、1〜3分ほど待ってから再読み込み（更新）してください。
2. スマートフォンでも同じURLで開けます。
3. このURLをお子さんに送れば、いつでもどこでもクイズに挑戦できます。
   - 📱 スマホでURLを開いたあと「ホーム画面に追加」しておくと、アプリのように使えます。

---

## 問題を追加・修正したくなったら

- 問題は `data` フォルダの中の `.json` ファイルにあります。
- 新しい時代を追加したら、`data/manifest.json` にもその時代を追記します。
- 修正したファイルを、同じリポジトリで再度「Add file → Upload files」して上書きすれば、
  数分後に自動で公開ページに反映されます。

（このあたりの更新作業は、お父さん＝あなたの代わりに私が手伝えます。お気軽にどうぞ。）

---

## こまったときは

| 症状 | 確認すること |
|---|---|
| ページが真っ白／404 | `index.html` がリポジトリの一番上にあるか。Pages の Branch 設定が正しいか。 |
| 問題が表示されない | `data` フォルダがアップロードされているか。`manifest.json` があるか。 |
| 画像が出ない | 画像はインターネット上のものを読み込みます。オフラインだと出ないことがあります（問題は解けます）。 |
| 反映されない | 公開には数分かかります。ブラウザの更新（Ctrl＋F5）も試してください。 |
